Project Idea

Overview

My project idea is to create an application for calorie counting on Windows 10 with hopes to bring it to other platforms, notably IOS and Android. Easy Counter is a calorie counter built around being customizable for the user while also keeping things simple. I find that with most calorie counters I have used there are lots of confusing features and bad UI's that can make them intimidating. With Easy Counter I want to fix this by keeping things simple in the UI department while also giving it the features I have noticed are missing in the calorie counters I rely on to track my eating.

Description

My product will be a Windows 10 desktop application named Easy Counter that allows you to count the calories of the foods and drinks you consume. Easy Counter will have all the features I consider to be at the center of any calorie counter along side all the ones I personally feel are necessary to make using Easy Counter a good user-friendly experience.

To start off with I want Easy Counter to have a UI with separate tabs for distinctly different sets of information and functionality so that the UI doesn't get muddled up with separate parts of the app on the same screen. The main tab will contain your daily caloric information represented with a large visual in the center of the screen. Alongside the daily calories visual will be buttons to access separate meal windows that you can add yourself to give order to your meals and snacks. In each of these sections you can add foods from a large list that comes with the app or you can add your own custom foods and enter the caloric or kilo joules value of that food per 100 grams.

One of the quality-of-life features that Easy Counter will have is being able to choose between kilo joules or calories and adding your own foods work in tandem with this. Most foods list their nutritional value in kilo joules and on Easy Counter you can enter that value off the product with the conversion between kilo joules to calories happening in the background so that you don't have to worry about doing it yourself. Not only can you add your own foods but also recipes or meals consisting of separate foods so that you can log your own home cooking.

One of the big problems I find calorie counters have is that they can become quite limited with restaurant foods especially when that restaurant doesn't list their foods nutritional information. Easy Counter will have foods that are the average of multiple different restaurants same food like fries. Foods from restaurants that do list their nutritional value will be separate because those will be more accurate for that individual food. Another handy function I have always desired in calorie counters is in relation to customizing orders from restaurants. If you get a cheeseburger with extra cheese and without tomato you will be able to do that in the app while still tracking an accurate nutritional value of the food.

The second tab of Easy Counter will be where you can customize your settings, preferences and information. The preferences section of this tab will be where you choose your app theme and choice of nutrition value those being calories or kilo joules. There may be more preferences necessary but for now those are the only that I think are important. The information surrounding the users actual weight loss are managed in this section. This information such as weight, weight goal and calorie budget not only help the app function but act as a way for the user to keep track of things like their own weight loss overtime. I plan to add more functionality, features and to make Easy Counter more complex over time but for now I see tools detailed in this description to be the most important in the first release version to be immediately valuable and usable.

Tools and Technologies

I will use Python as the primary tool in creating my application. Git and Github will be used for managing source control as well as a way to download the finished product. Since Easy Counter is a GUI app I chose to use a Python library named Tkinter since it was designed specifically for GUI apps and is great for beginners due to it's simplicity. With a GUI app comes the need for graphics and UI elements which I will create myself using Krita, a free and open source art application which I know I can rely on since it has served me countless times in the past. The website to go with the project will be made of static HTML CSS with hosting on Github Pages.

Skills Required

The most obvious skill required is sufficient knowledge in writing Python programs. Learning this will be easy, as Python is a simple syntax scripting language with a plethora of tutorials and teachings in various formats throughout the internet. There is also the Tkinter Python library which has a larger focus on being beginner friendly then other GUI libraries. I will need basic GitHub and Git skills but I already have sufficient knowledge of both to accomplish what I need. Lastly, Krita. While most art applications have a learning curve and sometimes and intimidating amount of features I am already familiar with the apps ins and outs as I have used it for years before hand.

Outcome

Assuming the project is successful I hope that my application makes calorie counting easier for users of calorie counter apps everywhere. Having an application that can fit a plethora of different eating situations whether that be home cooking, packaged food or restaurants in an easy to use app will remove that part of the stress of calorie counting and let you focus on your main goal. Instead of not being able to reliably count the calories on your highly customized McDonalds cheeseburger meal you will be able to edit it just as much in the app as you would at the restaurant. Having a calorie counter app that addresses all the other apps' shortcomings would encourage them to step up their game so that people don't have to dig through a large amount of calorie counters all with different features trying to settle on which one fits their needs the most.
